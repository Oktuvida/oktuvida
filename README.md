## A quick introduction before I overengineer this README

Hi. I'm Oktuvida.

I'm the kind of software engineer who likes systems that are testable, reproducible, automated, and slightly allergic to nonsense.

I learn the practical way: build things, break things, inspect the ashes, improve the architecture, repeat tomorrow.

## My default operating mode

- SDD, because vague vibes are not a software strategy.
- TDD, because I enjoy sleeping without wondering what production is about to do.
- Strict code quality, ideally enforced by automation so humans can spend less time arguing with lint and more time thinking.
- Everything-as-Code, because if the setup lives only inside somebody's memory, that is basically folklore.
- Everything-Reproducible, because future me deserves rights.
- Open-source first, because I like knowing how the machine works before I trust it with my soul.

## Current obsession

Right now I'm learning how to use AI agents to build products that are actually robust, secure, and optimized.

Not just "look, it generated code" impressive.

I mean "this thing survives contact with reality" impressive.

## Tiny manifesto, large consequences

```text
If it cannot be tested, I do not trust it enough.
If it can be automated, I do not want to babysit it manually.
If it is not reproducible, it is a future bug wearing a disguise.
If there is a good open-source alternative, I will probably investigate it.
```
